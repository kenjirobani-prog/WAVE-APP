# 波予報アプリ 仕様書 SPEC.md

## 1. コンセプト

**湘南特化・パーソナライズ波診断アプリ**

自分のレベル・ボード・好みの波サイズを登録しておくと、サイトを開いた瞬間に「今日あなたが行くべきスポット」を◎○△×でランキング表示してくれるWEBアプリ。

### 競合との差別化
- 波伝説・なみある → データを見てユーザーが自分で判断する設計
- 本アプリ → ユーザーの属性に合わせた答えを直接提示（パーソナライズ診断）
- 初心者でも波の専門用語なしで判断できるUI/UX
- 湘南エリアに特化し、スポットの特性を深く作り込む

## 4. 画面設計

### 4-1. オンボーディング（2ステップ）

ログイン不要。設定はlocalStorageに保存。「あとで設定する」でスキップ可能（デフォルト値：中級者・胸サイズで診断開始）。

**Step 1 — レベル・ボード**
- サーフィンレベル選択（初級者 / 中級者 / 上級者）
- 使用ボードの種類（ロングボード / ファンボード / ショートボード / フィッシュ）

**Step 2 — 波サイズ・スポット**
- 好みの波サイズ（〜腰 / 胸〜肩 / 頭 / オーバーヘッド）
- よく行くスポット（複数選択可）

### 4-2. トップ画面（おすすめランキング）

- ヘッダー：今日の概況サマリー（波・風・天気）
- 日程タブ：今日 / 明日 / 週末
- スポットカード一覧：◎○△×グレード + スコア + 理由タグ
- 「よく行くスポット」はバッジつきで上位固定表示
- ボトムナビ：波予報 / スポット / マイページ（3タブ）

### 4-3. スポット詳細画面

上から順に以下のセクション：

1. **グレード表示（◎○△×）**
2. **おすすめ理由テキスト**（日本語で平易に説明）
3. **現在の4指標カード**
   - 波の高さ（m + 体感サイズ表示）
   - 風（m/s + オフショア/サイド/オンショア）
   - うねりの方向（度数 + 正面/斜め/外れ）
   - 波の周期（秒 + うねりあり/なし）
4. **1時間ごと予報チャート（横スクロール）**
   - 波高バー（色で好みサイズ判定：緑=◎ / 青=○ / 薄青=△）
   - 風矢印（色でオフショア/サイド/オンショアを表現）
   - 時間軸（現在時刻を▲マークで強調）
5. **潮位バー**（現在位置 + 上げ/引きの動き）
6. **コミュニティレポート**
   - 直近レポートを表示
   - 「今日のレポートを書く」ボタン

## 5. 採点ロジック

### 5-1. 基本スコア（100点満点）

| 採点軸 | 配点 | 採点ルール |
|--------|------|-----------|
| 波の高さ | 30点 | 好みサイズ以上 → 30点 / −30cm以内 → 20点 / それ以下 → 5点 / フラット → 0点 |
| 風（向き×強さ） | 30点 | オフショア5m以下 → 30点 / オフショア5〜10m → 22点 / サイドオフ5m以下 → 18点 / サイド → 12点 / オンショア〜8m → 5点 / オンショア8m超 → 0点 |
| うねりの方向 | 20点 | 正面（±30°以内） → 20点 / 斜め（±30〜60°） → 13点 / やや外れ（±60°超） → 6点 / 背面 → 0点 |
| 潮（水位×動き） | 15点 | 80〜120cm + 上げ込み → 15点 / 80〜120cm + 引き → 10点 / 60〜80cm or 120〜150cm → 7点 / 60cm以下 → 3点 / 150cm超 → 3点 |
| 天気ボーナス | +5点 | 晴れ → +5点 / 曇り → +2点 / 雨 → 0点 |

### 5-2. レベル補正（±最大15点）

| 採点軸 | 初級者 | 中級者 | 上級者 |
|--------|--------|--------|--------|
| 波の高さ | オーバーヘッド以上で −10点 | 補正なし | 胸以下で −10点 |
| 風の強さ | オンショア強風で追加 −5点 | 補正なし | オフショア強風で +3点 |
| 潮 | 干潮時（60cm以下）で追加 −5点 | 補正なし | 補正なし |

### 5-3. グレード閾値

| グレード | スコア | 意味 |
|---------|--------|------|
| ◎ | 80点以上 | 今日は絶対行け |
| ○ | 60〜79点 | まあまあ楽しめる |
| △ | 40〜59点 | 微妙、他スポットも比較して |
| × | 39点以下 | 今日は見送り推奨 |

### 6-2. 初期スポットデータ（5スポット）

| スポット | ベストうねり方向 | 許容範囲 | 最適潮位 | 初心者向き度 |
|---------|----------------|---------|---------|------------|
| 鵠沼 | 南〜南南東（160〜190°） | ±40° | 80〜120cm | ⭐⭐⭐⭐⭐ |
| 辻堂 | 南（170〜190°） | ±35° | 80〜120cm | ⭐⭐⭐⭐ |
| 茅ヶ崎 | 南〜南西（190〜220°） | ±30° | 70〜110cm | ⭐⭐⭐ |
| 大磯 | 南南西（200〜230°） | ±30° | 70〜110cm | ⭐⭐ |
| 七里ヶ浜 | 南東〜南（140〜180°） | ±25° | 60〜100cm | ⭐ |

### 6-3. スポット管理方法

- Firestore に保存し、管理画面（`/admin`）からブラウザで編集可能
- Firebase Auth で管理画面を保護（管理者のみアクセス可）
- `isActive: false` で削除せず非表示にできる
- スポット追加・編集・並び替え・無効化をCRUD画面で対応

## 8. APIアダプター設計

将来の有料API切り替えに備え、アダプターパターンで実装する。

### 8-1. 共通インターフェース

```typescript
// lib/wave/types.ts
interface WaveCondition {
  spotId:       string
  timestamp:    Date
  waveHeight:   number    // メートル
  wavePeriod:   number    // 秒
  swellDir:     number    // 度（0-360）
  windSpeed:    number    // m/s
  windDir:      number    // 度
  tideHeight:   number    // cm
  tideMovement: 'rising' | 'falling' | 'slack'
  weather:      'sunny' | 'cloudy' | 'rainy'
}

interface WaveAdapter {
  getConditions(spotId: string, date: Date): Promise<WaveCondition[]>
  getForecast(spotId: string, days: number): Promise<WaveCondition[]>
}
```

### 8-2. API切り替え方法

```bash
# .env.local
WAVE_API_PROVIDER=open-meteo  # stormglass / surfline に変えるだけ
```

```typescript
// lib/wave/waveService.ts
const adapter = {
  'open-meteo': () => import('./adapters/openMeteo'),
  'stormglass': () => import('./adapters/stormGlass'),
  'surfline':   () => import('./adapters/surfline'),
}[process.env.WAVE_API_PROVIDER]
```
